# 赫兹 Hertz Demo Script

## Demo Goal

在 3 分钟内让评委看到这不是通用 AI 助手，而是面向 ASD 家庭高压照护场景的闭环支持系统。

## Fixed Storyline

- 家庭画像：8 岁 ASD 儿童，短句表达，对声音敏感，常见高摩擦场景是过渡和睡前。
- 今日状态：睡眠不足，家长压力高，今天放学回家后关屏和换衣容易升级。
- 演示目标：展示系统如何从签到判断风险，再给出高摩擦支持、训练任务和复盘学习。

## 3-Minute Flow

### 1. Home / 今日签到

- 停留时长：30 秒
- 要做的动作：打开首页，点击“开始今日签到”
- 要说的话：
  - “家长不用先判断今天该练什么，先按真实状态签到。”
  - “系统会基于最近签到和事件记录，自动判断今天是先减负、先稳住，还是可以推进训练。”

### 2. 今日聚焦

- 停留时长：20 秒
- 要做的动作：完成签到后展示首页的“今天最重要的一件事”
- 要说的话：
  - “这里不是长篇建议，只保留今天最该做的一件事。”
  - “高压日优先减负，不会硬推训练任务。”

### 3. Scripts / 高摩擦支持

- 停留时长：60 秒
- 要做的动作：进入 `Scripts` 页，选择“过渡”或“关屏”预设，生成支持方案
- 要说的话：
  - “这里针对的是家长最容易崩的现场，不是事后总结。”
  - “系统会结合孩子禁忌、感官状态、照护者压力，生成三步动作、现场话术和退场方案。”
  - “如果输入里出现高风险信号，系统不会继续生成普通建议，而是直接进入安全阻断。”

### 4. Plan / 训练工作台

- 停留时长：40 秒
- 要做的动作：进入 `Plan` 页，打开一个优先能力点和今日任务
- 要说的话：
  - “当家庭状态允许推进时，系统才给低负担训练任务。”
  - “训练不是固定模板，而是根据画像、最近签到、复盘反馈动态调整难度和阶段。”

### 5. Review / 反馈学习

- 停留时长：20 秒
- 要做的动作：展示一次“部分完成”或“孩子过载”的反馈结果
- 要说的话：
  - “家长只需要反馈是否完成、孩子反应如何，系统就会自动调整下次策略。”
  - “这一步体现产品不是一次性生成，而是会逐步学会这个家庭的有效做法。”

### 6. Weekly Report / 周报

- 停留时长：10 秒
- 要做的动作：展示周报摘要
- 要说的话：
  - “最后把每天的签到、事件和反馈汇总成周报，让家长和老师能看到哪些方法真的有效。”

## One-Line Positioning

`赫兹 Hertz 不是一个给家长聊天的 AI，而是一个把“签到、风险判断、现场支持、训练推进、复盘学习”串起来的家庭照护操作系统。`

## Demo Rules

- 不在演示里讲模型参数、框架名或数据库细节，先讲家长价值。
- 每一页只强调一个判断点，不同时讲功能和实现。
- 只走一条固定路径，不现场自由探索。
- 遇到安全阻断时，强调“宁可不生成，也不输出危险建议”。
