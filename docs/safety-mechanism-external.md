# 赫兹 Hertz 安全机制说明

本文面向家长、合作方与评审场景，用于说明赫兹 Hertz 如何在照护支持场景中控制风险、降低不当建议输出的概率。本文描述基于当前产品实现，重点是安全边界与防护机制，不构成医疗建议或临床承诺。

## 1. 我们的安全边界

赫兹 Hertz 不是医疗诊断系统，也不提供处方、替代医生判断或危机干预机构。产品的定位是家庭照护支持工具，帮助家长在高压力、强摩擦的日常场景中获得更可执行、更低刺激、更符合家庭约束的支持建议。

这意味着，系统从设计上避免输出以下类型内容：

- 诊断结论
- 用药建议
- 医疗处置方案
- 在高风险场景下继续推进普通行为建议

当输入中出现明显高风险信号时，系统不会继续生成常规脚本，而会直接进入安全阻断与求助引导流程。

## 2. 核心安全思路

赫兹 Hertz 的安全机制不是依赖单次模型回答“自己变谨慎”，而是通过多层约束，把危险建议尽量拦在输出前。整体原则是：

- 先识别风险，再决定是否允许继续生成
- 先检查个体禁忌，再决定建议是否可执行
- 先确认有证据支撑，再允许输出完整方案
- 如果条件不满足，宁可降级、缩短或阻断，也不继续给高不确定性建议

换句话说，产品优先追求“避免明显危险输出”，而不是“任何情况下都给出完整建议”。

## 3. 具体怎么做

### 3.1 高风险信号阻断

系统会对用户输入中的高风险信号做优先检测。包括：

- 用户主动勾选高风险状态
- 输入文本中出现明显危险关键词
- 场景已经接近自伤、他伤或失控攻击等高风险状态

一旦命中高风险规则，系统不会继续输出普通计划或沟通脚本，而是进入安全阻断模式。此时产品只保留以下几类内容：

- 现在先做什么
- 现在不要做什么
- 一句可以立刻说出口的话
- 退场步骤
- 求助建议
- 联系人消息模板

这样做的目的，是把输出从“继续推进任务”切换为“先保安全、先降刺激、先求助”。

### 3.2 个体禁忌冲突校验

产品允许家庭在档案中填写孩子的禁忌和明确不能做的事，例如：

- 不可突然触碰
- 不可大声催促
- 不要连续追问原因

系统生成建议后，会再做一次禁忌冲突检查。如果建议内容与家庭档案中的禁忌相冲突，例如明明标记了“不可触碰”，建议里却出现“抱住孩子”“强拉离开”，系统会直接阻断该建议，并改为提示原因和替代做法。

这一步的意义在于，安全不是抽象的，而是以每个家庭的具体边界为准。

### 3.3 证据约束与引用要求

系统不是直接自由生成，而是先从内部策略卡和知识单元中检索相关支持内容，再组织成建议输出。每一份计划、脚本或高摩擦支持方案，都要求带有对应的引用标记。

如果缺少必要引用，系统会视为证据不足并阻断输出，而不是继续给出看起来完整、但缺乏支撑的建议。

这套设计的目标，是降低“听起来合理但没有依据”的生成风险。

### 3.4 多阶段审查，而不是一步出答案

系统在生成建议前后，会经过多个阶段，而不是一次性直接给结论。简化理解，可以分为四步：

1. 先理解当前场景和风险状态
2. 再召回与当前家庭、场景、强度匹配的支持内容
3. 生成候选建议
4. 对候选建议做安全审查和证据审查

如果安全审查不通过，结果会被阻断；如果证据不充分，结果会被降级或拒绝输出。这样做是为了减少“模型一口气编完整方案”的不稳定性。

### 3.5 低刺激与退场优先

在高摩擦场景下，系统的默认倾向不是强推任务完成，而是优先建议：

- 降低刺激
- 缩短指令
- 减少沟通人数
- 暂停原任务
- 必要时转入退场或交接

如果现场支持不足、照护者已经过载，系统会进一步把建议压缩成更轻、更短、更易执行的一步，避免在高压状态下输出复杂要求。

## 4. 用户会看到哪些安全保护

从用户体验上，安全机制主要体现在以下几类结果：

- 高风险时不显示普通脚本，而显示安全阻断卡
- 冲突建议不会直接下发，而会提示为什么不安全
- 建议会明确列出“不要做什么”
- 建议会提供一句可立即使用的话术
- 必要时提示切换低刺激模式
- 必要时提示联系支持者、学校或本地紧急资源

这意味着，产品的目标不是在所有情况下都“给答案”，而是在不适合继续推进时，清楚告诉用户应该先停下来、先保安全。

## 5. 我们为什么认为这比普通聊天机器人更安全

相较于通用聊天机器人，赫兹 Hertz 在安全上做了几层额外约束：

- 有明确的非医疗边界，不把自己包装成诊断工具
- 有高风险阻断链路，而不是高风险时继续自由生成
- 有个体禁忌校验，而不是只看通用常识
- 有证据引用要求，而不是无依据输出
- 有降级与退场机制，而不是默认把任务做到底

这些设计不能保证“永远正确”，但可以显著降低明显危险、明显不适配、明显无依据建议的输出概率。

## 6. 当前仍然保留的限制

出于真实和负责的原则，我们也明确说明当前限制：

- 产品不是临床诊断系统
- 不能替代医生、治疗师、学校危机流程或本地急救系统
- 高风险识别仍依赖规则、输入质量和场景表达，不能视为百分之百覆盖
- 安全机制的目标是降低风险，不是提供医疗保证
- 对地域化热线、医院、机构资源的覆盖仍需结合实际部署补充

因此，在出现持续自伤、他伤、武器、窒息风险、严重失控或其他人身安全威胁时，应立即联系当地急救、危机干预资源或线下专业人员，而不是继续等待系统生成更多建议。

## 7. 一段适合对外简述的话

赫兹 Hertz 不是做诊断和处方的医疗系统，而是一个带安全护栏的家庭照护支持工具。我们通过高风险阻断、个体禁忌校验、证据引用要求、多阶段安全审查和低刺激退场机制，尽量减少危险建议输出。在高风险场景下，系统不会继续生成普通建议，而是直接切换到安全阻断和求助引导。
